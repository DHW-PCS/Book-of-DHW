# DHW Inf 的现在与未来

不知不觉间，上次版本更新说明发布已经是近两年的事情。而此次更新说明发布后，也没有人知道下一次版本更新说明是什么时候。因此，与其说明今年 Inf 服务器更新了什么，开度署署长更想借此次机会，向各位成员和玩家就服务器版本更新的话题补充过去没有机会说明的内容，并表达未来的愿景。

开度署曾经在 [1.20 更新快速说明](inf-1.20-update.md#总结)中将 1.22 定为下次更新的目标。然而，计划赶不上变化。在 Minecraft 1.20.4 之后，Mojang 更改了游戏的更新策略：从以前的年度主要更新变成一年多次的[小更新](https://minecraft.wiki/w/Game_drop)。到目前为止，于2024年6月13日发布的[“棘巧试炼”（1.21）](https://zh.minecraft.wiki/w/%E6%A3%98%E5%B7%A7%E8%AF%95%E7%82%BC)是游戏的最后一个主要更新。

在游戏的主要内容随着小版本更新不断加入游戏的情况下，开度署已经预感到 1.22 很可能不会到来，并于2025年8月2日将 DHW Inf 升级到 1.21.7。

如今，这个预感已变成每个 Minecraft 玩家都需要时间接受的现实。2025年12月2日，Mojang 发布了[新的版本策略](https://www.minecraft.net/en-us/article/minecraft-new-version-numbering-system)，2026年开始，新版本将利用类似[语义化版本](https://semver.org/lang/zh-CN/)的逻辑，使用发布年份、小更新序号和补丁号定义版本。这意味着 1.21.11 的下一个版本既不是 1.21.12，也不是 1.22，而是 26.1。

在越来越多服务器端模组已经跟不上快节奏小更新的情况下，本服版本更新已经面临极大的不确定性；而玩家和模组开发者需要一段时间适应的新版本编号，势必也会对未来的更新造成不小的影响。

DHW Inf 至今七年多的历史，充分展现其不断发展和进化的精神。因此，开度署在此感谢让服务器得以持续的玩家和其他管理。

为了应对未来的挑战，开度署署长决定重新设计本服更新规则，让其最大程度地贴近官方更新节奏：

1. 开度署的服务器版本升级目标为每年1-2次。“升级”是指年度或小更新（drop）版本号提升的更新，例如从 26.1 更新到 26.2 或 27.1。
2. 管理将通过 [inf-mods](https://github.com/dhw-pcs/inf-mods) 项目自动检查和部署服务器模组，并定期剔除超过半年未更新的模组。不受本项目管理的模组将在下次升级时由管理从服务器中删除。
3. 管理可能会在无人游玩时更新补丁版本，例如从 26.1 更新到 26.1.1。管理不会就此类更新通知玩家，除非玩家需要更新客户端。

相信在新的规则和 inf-mods 项目的帮助下，服务器能够在维护压力几乎保持不变的情况下实现更快速的更新，对于玩家和管理而言都是双赢的局面。

与此同时，开度署署长希望了解玩家和其他管理的想法。正如 Mojang 对 Minecraft 的游戏更新策略进行多次修改一样，上述规则也有可能根据情况变化而有所改动。如有任何意见或建议，请直接提出。

祝游戏愉快！
